Decisions must be made to create a successful project. And it's important to document the decisions, so that things make sense and can be confindently changed as the circustances change.

To document them, here we use a type of document called Architectural Decision Record (ADR). You can read why, when and how in the [seminal article](https://cognitect.com/blog/2011/11/15/documenting-architecture-decisions) by Michael Nygard. Here a different file name style is used, for readability, but that's not important.

By now it's not well done, because the project is being developed by just one person, with no previous experience creating a electronics product. But the goal is there.
